# 复盘记录

## 问题：财务数据读取不完整

**时间：** 2026-03-19
**项目：** finance-ledger

### 问题描述
用户询问昨天（2026-03-18）的财务记录，我只读取到一条数据（咖啡 6元），但实际上有3条记录：
- 咖啡 ￥6.00
- 午饭 ￥26.00  
- 晚饭 ￥22.00

### 根本原因
**偷懒 + 粗心**

1. **使用了错误的读取方式**
   - 用了 `offset: 195` + `limit: 10` 固定偏移量读取
   - 没有使用搜索或过滤确保获取完整数据

2. **没有验证数据完整性**
   - 看到一条数据就停止检查
   - 没有确认是否还有同日期的其他记录

3. **缺乏健壮性考虑**
   - 假设数据在固定位置
   - 没有处理数据追加导致的偏移变化

### 正确做法
```powershell
# 应该使用搜索获取完整数据
Select-String -Path "2026.md" -Pattern "2026-03-18"
```

或

```powershell
# 读取更大范围并过滤
Get-Content "2026.md" | Where-Object { $_ -match "2026-03-18" }
```

### 教训
1. **不要假设数据位置固定**
2. **使用搜索/过滤而非固定偏移**
3. **验证数据完整性（计数、范围）**
4. **看到部分数据时要警觉是否完整**

### 改进措施
- 财务数据查询必须使用日期搜索
- 返回结果前确认记录数量是否合理
- 不确定时扩大读取范围再过滤
